# 会话记录格式规范

> 定义各采集器输出的 Markdown 格式标准

---

## 一、设计原则

1. **人类可读**：用户可直接查看了解今天做了什么
2. **结构清晰**：统一的标题、列表、时间格式
3. **信息完整**：包含会话基本信息、输入、输出、文件修改
4. **易于汇总**：便于 main.py 提取关键信息生成日报

---

## 二、文件命名规范

```
<采集器名称>会话记录.md

示例：
- ClaudeCode会话记录.md
- VSCode会话记录.md
- IDEA会话记录.md
- System时间线.md
```

## 七、格式规范要点

### 7.1 时间格式
- 统一使用：`YYYY-MM-DD HH:MM:SS` 或 `HH:MM:SS`
- 时长格式：`2小时15分` 或 `2h15m`

### 7.2 表格规范
- 使用 Markdown 表格语法
- 对齐方式：左对齐
- 必要时使用表情符号增强可读性

### 7.3 代码块
- 使用语法高亮：\`\`\`python
- 文件路径使用行内代码：\`config.py\`

### 7.4 统计数据
- 每个文档末尾都有"附录：统计数据"
- 便于 main.py 提取关键指标
