## 四、生成逻辑

### 4.1 数据来源

```python
# main.py 中的生成逻辑
def generate_daily_report(output_dir: Path, summaries: list, today: datetime):
    """
    从各采集器的 sessions.json 汇总生成日报
    
    Args:
        output_dir: 输出目录（.../2026-03-25/）
        summaries: 各采集器的摘要信息
        today: 今天日期
    """
    
    # 1. 收集所有 sessions.json
    all_sessions = []
    for summary in summaries:
        json_file = output_dir / summary['output_subdir'] / 'sessions.json'
        if json_file.exists():
            with open(json_file, 'r', encoding='utf-8') as f:
                data = json.load(f)
                all_sessions.extend(data.get('sessions', []))
    
    # 2. 按项目分组
    projects = group_by_project(all_sessions)
    
    # 3. 计算统计数据
    stats = calculate_statistics(all_sessions, summaries)
    
    # 4. 生成 Markdown
    md_content = render_report(projects, stats, today)
    
    # 5. 写入文件
    report_file = output_dir / "今日工作总结.md"
    with open(report_file, 'w', encoding='utf-8') as f:
        f.write(md_content)
```

### 4.2 统计指标

```python
def calculate_statistics(sessions, summaries):
    """计算核心指标"""
    return {
        'total_duration': sum(s.get('duration_minutes', 0) for s in sessions),
        'total_files': sum(s.get('files_modified', []).__len__() for s in sessions),
        'session_count': len(sessions),
        'project_count': len(set(s.get('project') for s in sessions)),
        'ai_usage_hours': sum(
            s.get('duration_minutes', 0) 
            for s in sessions 
            if s.get('collector') == 'claude_code'
        ) / 60,
    }
```

## 五、格式规范要点

### 5.1 标题层级
- `#` 一级标题：日报标题（含日期）
- `##` 二级标题：主要章节（概览、成果、分析等）
- `###` 三级标题：项目名称、子章节
- `####` 四级标题：细节内容

### 5.2 表情符号使用
- 📊 统计数据
- 🎯 目标成果
- 📈 趋势分析
- 🔧 技术相关
- 💡 思考建议
- ✅ 已完成
- 🔄 进行中
- ❌ 未完成

### 5.3 数据呈现
- 优先使用表格展示结构化数据
- 重要数字加粗：**7小时45分**
- 百分比保留一位小数：39.5%

### 5.4 时间表达
- 统一格式：`2026年03月25日（星期三）`
- 时长格式：`2小时15分` 或 `2h15m`
