## 一、数据源技术约束概览

> **详细数据存储机制** 详见：[调研各组件的数据存储机制.md](./调研各组件的数据存储机制.md)

### 1.1 采集器约束矩阵

| 采集器 | 难度 | 数据可读性 | 权限要求 | 数据格式 | 实现优先级 |
|--------|------|------------|----------|----------|------------|
| Claude Code | ⭐ 简单 | ✅ 完全可读 | 无 | JSONL 明文 | P0 |
| VSCode | ⭐⭐ 中等 | ✅ 可读 | 无 | 目录 + JSON | P1 |
| IDEA | ⭐⭐ 中等 | ✅ 可读 | 无 | XML + 文件系统 | P1 |
| CodeBuddy | ⭐⭐ 中等 | ✅ SQLite 可读 | 无 | SQLite + JSON | P1 |
| System | ⭐⭐⭐ 困难 | ⚠️ 部分可读 | 辅助功能权限 | 系统日志 | P2 |

### 1.2 数据源路径速查

| 采集器 | 数据源路径 | 关键文件 |
|--------|-----------|----------|
| Claude Code | `~/.claude/` | `history.jsonl`, `projects/<project>/<session>.jsonl` |
| VSCode | `~/Library/Application Support/Code/User/History/` | 文件快照目录 |
| IDEA | `~/Library/Application Support/JetBrains/<Product>/` | `recentProjects.xml`, `recentFiles.xml` |
| CodeBuddy | `~/Library/Application Support/CodeBuddy/storage/` | `codebuddy.db` (SQLite), `sessions/`, `messages/` |
| System | `/var/log/system.log` | 系统日志 |

### 1.3 核心技术难点

#### Claude Code
- ⚠️ **路径转换**：项目路径 `/` 需替换为 `-` 才能定位目录
- ⚠️ **大文件处理**：单个会话可能包含大量 token，需流式读取
- ⚠️ **并发写入**：用户正在使用时可能存在文件锁定

#### VSCode
- ⚠️ **路径哈希**：文件路径通过哈希映射，需逆向查找
- ⚠️ **增量识别**：同一文件可能多次修改，需去重
- ⚠️ **工作区识别**：需从文件路径推断所属工作区

#### IDEA
- ⚠️ **多产品支持**：IDEA、WebStorm、PyCharm 等多个产品
- ⚠️ **版本差异**：不同版本目录结构可能不同
- ⚠️ **变量解析**：XML 中的 `$USER_HOME$` 等变量需解析

#### System
- ❌ **权限要求**：需用户手动授权辅助功能权限
- ⚠️ **后台运行**：需守护进程持续监听
- ⚠️ **隐私问题**：窗口标题可能包含敏感信息

#### CodeBuddy
- ❌ **数据加密**：`entry_*.info` 文件加密，无法解析
- ❌ **缺少文档**：官方未公开数据格式
- ⚠️ **版本更新**：数据格式可能随版本变化

### 2.2 权限申请策略

#### 文件访问权限
```python
import os
from pathlib import Path

def check_file_access(path: Path) -> bool:
    """检查文件访问权限"""
    try:
        if path.exists():
            return os.access(path, os.R_OK)
        return False
    except PermissionError:
        return False
```

#### 辅助功能权限（System 采集器）
```python
# macOS 辅助功能权限检查
import subprocess

def check_accessibility_permission() -> bool:
    """检查辅助功能权限"""
    try:
        result = subprocess.run(
            ['osascript', '-e', 'tell application "System Events" to get name of processes'],
            capture_output=True,
            text=True
        )
        return result.returncode == 0
    except:
        return False
```

### 3.3 大文件处理策略

```python
# 流式读取大文件
def read_large_jsonl(file_path: Path, batch_size: int = 1000):
    """流式读取大 JSONL 文件"""
    batch = []
    with open(file_path, 'r', encoding='utf-8') as f:
        for line in f:
            batch.append(json.loads(line))
            if len(batch) >= batch_size:
                yield batch
                batch = []
    if batch:
        yield batch
```

## 五、安全约束

### 5.1 敏感信息识别

```python
# 敏感信息模式
SENSITIVE_PATTERNS = [
    r'api[_-]?key["\s:]+["\']?[\w-]+',
    r'password["\s:]+["\']?[\w-]+',
    r'secret["\s:]+["\']?[\w-]+',
    r'token["\s:]+["\']?[\w-]+',
    r'AK[A-Z]{2}[\w]{16}',  # 阿里云 AccessKey
    r'sk-[\w]{48}',  # OpenAI API Key
    r'[\w\.-]+@[\w\.-]+\.\w+',  # 邮箱地址
    r'\b\d{17}[\dXx]\b',  # 身份证号
    r'\b1[3-9]\d{9}\b',  # 手机号
]
```

### 5.2 脱敏策略（预留）

```python
import re

def sanitize_content(content: str, patterns: list = None) -> str:
    """
    内容脱敏处理
    
    Args:
        content: 原始内容
        patterns: 敏感信息模式列表
    
    Returns:
        脱敏后的内容
    """
    if patterns is None:
        patterns = SENSITIVE_PATTERNS
    
    for pattern in patterns:
        content = re.sub(pattern, '[REDACTED]', content, flags=re.IGNORECASE)
    
    return content
```

## 六、错误处理约束

### 6.1 错误分类

| 错误类型 | 处理策略 | 日志级别 |
|----------|----------|----------|
| 文件不存在 | 跳过采集器，继续执行 | WARNING |
| 权限不足 | 记录错误，提示用户 | ERROR |
| 数据格式错误 | 尝试解析，失败则跳过 | WARNING |
| 内存不足 | 释放资源，优雅退出 | CRITICAL |
| 网络错误 | 离线模式，使用缓存 | WARNING |

### 6.2 错误恢复机制

```python
from functools import wraps
import logging

logger = logging.getLogger(__name__)

def safe_collector(func):
    """采集器安全装饰器"""
    @wraps(func)
    def wrapper(*args, **kwargs):
        try:
            return func(*args, **kwargs)
        except FileNotFoundError as e:
            logger.warning(f"文件不存在: {e}")
            return False
        except PermissionError as e:
            logger.error(f"权限不足: {e}")
            return False
        except json.JSONDecodeError as e:
            logger.warning(f"JSON 解析错误: {e}")
            return False
        except Exception as e:
            logger.error(f"未知错误: {e}", exc_info=True)
            return False
    return wrapper
```
