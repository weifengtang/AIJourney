> **验收要求**：每个里程碑完成后必须进行**数据验证**，包括：
> 1. 路径/数据格式验证（确保可访问）
> 2. 实际运行测试（使用真实数据）
> 3. 报告内容验证（检查消息数、Token统计等）
> 4. 格式规范验证（符合`规范格式/*.md`要求）

> **注**：M4 CodeBuddy 采集器修复了以下问题（v2.1.0）：
> - 路径修正：`CodeBuddy/storage` → `CodeBuddyExtension/Data`
> - 格式修正：SQLite → JSON 文件
> - 消息解析：`message` 字段是 JSON 字符串，需二次解析
> - Token统计：从 `requests[].usage` 获取

## 三、关键决策

| 决策项 | 方案 | 理由 | 决策时间 |
|--------|------|------|----------|
| 输出格式 | JSON + Markdown | 机器可读 + 人类可读 | 2026-03-25 |
| 扩展机制 | 插件化架构 | 支持自定义采集器 | 2026-03-25 |
| 容错策略 | 采集器隔离 | 单个失败不影响其他 | 2026-03-25 |
| 敏感信息 | 暂不实现 | 先跑通流程 | 2026-03-25 |
| System 采集器 | 降级为 P2 | 需要权限，难度大 | 2026-03-25 |
