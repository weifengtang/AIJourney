# AIJourney - 记录你的 AI 编程成长之旅

> 文档版本：v1.0
> 更新时间：2026-03-25
> 项目目标：自动采集 AI 编程工具会话数据，多维度汇总成长轨迹

---

## 一、项目概述

### 1.1 核心目标
自动记录程序员与 AI 编程工具的协作过程，每日自动生成工作汇总，帮助复盘成长。

### 1.2 支持的数据来源
| 采集器 | 优先级 | 数据源 | 状态 |
|--------|--------|--------|------|
| Claude Code | P0 | `~/.claude/` JSONL 文件 | 可直接采集 |
| VSCode | P1 | 本地历史文件 | 需调研 |
| IDEA | P1 | JetBrains 本地数据 | 需调研 |
| 系统窗口 | P2 | macOS 窗口历史 | 需权限 |
| CodeBuddy | P3 | `~/.codebuddy/` 加密数据 | 需解密方案 |

### 1.3 运行方式
- 命令行：`python main.py`
- 输出：按日期生成归档目录，包含 JSON（机器可读）和 Markdown（人类可读）两种格式

---

## 二、核心设计思想

### 2.1 插件化采集器架构
- **基类**：所有采集器继承 `BaseCollector`，必须实现 `collect()` 和 `_generate_markdown()` 方法
- **插件注册**：通过装饰器 `@register_collector(name)` 自动注册新采集器
- **统一输出**：每个采集器输出 `sessions.json` 和 `{名称}会话记录.md`

### 2.2 数据格式
每个会话包含：
- 会话ID、项目路径、起止时间、用户提问、修改文件、会话摘要

### 2.3 主流程
1. 按优先级执行所有已启用的采集器
2. 每个采集器独立执行，失败不影响其他采集器
3. 汇总所有采集结果，生成今日工作总结

### 2.4 配置管理
- 在 `config.py` 中可配置：
  - 路径、日期格式
  - 启用/禁用各个采集器
  - 设置执行优先级

## 四、设计原则

### 4.1 容错设计
- 单个采集器失败不影响整体运行
- 完整日志记录，便于排查问题

### 4.2 预留扩展
- 预留性能控制、数据保留、增量采集等功能空间
- 暂不实现敏感信息脱敏，先跑通核心流程
