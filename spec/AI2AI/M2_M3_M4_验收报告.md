## 三、M4: CodeBuddy 插件完成 ✅

### 验收标准
- [x] 可生成 JSON + Markdown
- [x] 跑通全流程

### 实现功能

> **重要修正**：原以为 CodeBuddy 使用 SQLite 数据库，实际验证发现使用 JSON 文件

#### 数据采集流程（实际验证后修正）
```
1. 遍历 CodeBuddyExtension/Data/<user-id>/
   ↓
2. 遍历 VSCode/JetBrains 目录
   ↓
3. 读取 history/<workspace>/index.json（会话列表）
   ↓
4. 读取 <session-id>/index.json（消息列表 + Token统计）
   ↓
5. 读取 messages/<msg-id>.json（消息内容）
   ↓
6. 解析嵌套 JSON（message 字段是 JSON 字符串）
   ↓
7. 从 requests[].usage 获取 Token 统计
   ↓
8. 生成 SessionData 对象
```

#### 核心特性

| 特性 | 说明 |
|------|------|
| ~~数据库查找~~ | ~~SQLite 数据库~~ ✅ 修正：JSON 文件 |
| 路径修正 | `CodeBuddy/storage` → `CodeBuddyExtension/Data` |
| 格式修正 | SQLite → JSON 文件 |
| 消息解析 | `message` 字段是 JSON 字符串，需二次解析 |
| Token统计 | 从 `requests[].usage` 获取，非 messages |

#### 核心方法

| 方法 | 功能 |
|------|------|
| `_load_workspace_sessions` | 加载工作区的会话 |
| `_load_session_detail` | 加载会话详细内容 |
| `_load_message` | 加载单条消息（解析嵌套JSON） |
| `_parse_iso_datetime` | 解析 ISO 时间格式 |

#### 发现的问题及修复

| 问题 | 原因 | 修复 |
|------|------|------|
| 路径不存在 | 配置写错目录名 | `CodeBuddy/storage` → `CodeBuddyExtension/Data` |
| 消息内容为空 | 读错字段 | 解析 `message` JSON字符串 |
| Token统计为0 | 找错数据位置 | 从 `requests[].usage` 累加 |

#### 版本信息
- **当前版本**：v2.1.0
- **版本历史**：v1.0.0（空实现）→ v2.0.0（SQLite方案，失败）→ v2.1.0（JSON方案，成功）
