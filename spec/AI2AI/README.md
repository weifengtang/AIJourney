# AI2AI 文档说明

## 概述

本目录下的 AI2AI 文档是 AI 协作开发文档，由 AI 维护，用于描述当前工程的系统状态、架构设计和数据协议，为 AI 编码协作提供清晰的技术规范和系统认知。

**定位**：
- **职责**：对当前代码状态的结构化总结
- **维护者**：AI（完全自主）
- **用途**：AI 在任务规划和编码前建立对工程的系统认知

---

## 文档清单

| 文档 | 用途 | 维护者 | 更新时机 |
|------|------|--------|----------|
| `需求方案调研.md` | 保存需求探索中的技术方案、调研历史（含数据存储机制调研） | AI | 需求探索时 |
| `概要设计.md` | 项目整体架构、组件技术设计 | AI | 架构设计时 |
| `项目实施计划.md` | 详细任务分解、进度跟踪、依赖关系 | AI | 每步完成后 |
| `协议和数据.md` | 数据格式、接口协议定义 | AI | 协议变更时 |
| `README.md` | 本文件，说明 AI2AI 文档的用途和使用方法 | AI | 文档结构变更时 |

---

## AI 工作流程

### 任务开始前

```
1. 读取 AI2AI/概要设计.md（理解架构）
   ↓
2. 读取 AI2AI/项目实施计划.md（理解进度）
   ↓
3. 读取 AI2AI/协议和数据.md（理解数据格式）
   ↓
4. 开始编码
```

### 任务完成后

```
1. 使用 maintain-ai2ai skill
   ↓
2. 校验 AI2AI 文档一致性
   ↓
3. 更新相关文档
   ↓
4. 确保文档与代码同步
```

---

## 文档使用原则

1. **AI 维护原则**: AI2AI 文档由 AI 完全维护，人类开发者不应直接修改这些文档
2. **一致性原则**: 所有 AI2AI 文档应保持内容一致，相互引用时应保持同步更新
3. **实时更新原则**: 当代码结构发生变更时，AI 应立即更新对应的 AI2AI 文档
4. **完整性原则**: 文档应尽可能完整地描述系统的各个模块和组件
5. **标准化原则**: 文档格式和内容组织应遵循统一的标准

---

## 文档更新时机

以下情况需要更新 AI2AI 文档：

### 需求方案调研.md
- 探索新的技术方案时
- 调研新的数据源时
- 记录技术决策历史时
- 更新数据存储机制调研时

### 概要设计.md
- 项目架构设计时
- 新增核心模块时
- 技术选型变更时
- 重构架构时

### 项目实施计划.md
- 完成任务步骤时
- 新增任务项时
- 任务状态变更时
- 发现阻塞问题时
- 依赖关系变更时

### 协议和数据.md
- 定义新的数据格式时
- 修改接口协议时
- 新增数据结构时
- JSON/Markdown 格式变更时

## 文档内容标准

### 概要设计.md 应包含

```markdown
# 概要设计 - [项目名]

> 维护者：AI（自动更新）
> 最后更新：YYYY-MM-DD
> 文档用途：项目整体架构、组件技术设计

## 一、项目概述
- 项目定位
- 核心价值

## 二、整体架构
- 架构图
- 核心模块

## 三、数据流设计
- 数据采集流程
- 数据结构定义

## 四、核心组件设计
- 各组件详细设计

## 五、技术选型
- 核心依赖
- 可选依赖
```

### 项目实施计划.md 应包含

```markdown
# 项目实施计划 - AI2AI 层

> 维护者：AI（自动更新）
> 最后更新：YYYY-MM-DD
> 文档用途：详细任务分解、进度跟踪、依赖管理

## 一、项目概览
- 基本信息
- 阶段进度总览

## 二、Phase N: [阶段名称]
### 任务清单
- 任务表格（优先级、预估时间、状态、完成时间）
- 验收标准

## 三、依赖关系
- 任务依赖图

## 四、质量保证
- 代码质量标准
- 功能质量标准

## 五、下一步行动
- 立即行动项
- 待确认问题
```

## 与 Me2AI 层的关系

```
Me2AI 层（输入）           AI2AI 层（过程 & 输出）
├─ 需求描述.md ────────→   ├─ 需求方案调研.md
├─ 任务规划.md ────────→   ├─ 概要设计.md
└─ 技术约束.md ────────→   ├─ 项目实施计划.md
                           └─ 协议和数据.md
```

**单向依赖原则**：
- AI2AI 依赖 Me2AI（读取需求约束）
- Me2AI 不依赖 AI2AI（保持独立性）
