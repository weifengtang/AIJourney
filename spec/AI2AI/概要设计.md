## 一、项目概述

### 1.1 项目定位
**AIJourney** 是一个记录 AI 编程成长之旅的工具，通过采集多个 AI 编程助手的会话数据，生成结构化的工作日报，让每一段 AI 协作都被永久记录。

### 1.2 核心价值
- 自动化：无需手动记录，自动采集工作痕迹
- 结构化：JSON + Markdown 双格式输出，机器可读 + 人类可读
- 可扩展：插件化架构，支持自定义采集器
- 容错性：采集器隔离，单个失败不影响其他

## 三、数据流设计

### 3.1 数据采集流程

```
用户运行 python main.py [--period daily|weekly|monthly|yearly]
        ↓
加载配置（config.py）
        ↓
加载所有采集器（collectors/__init__.py）
        ↓
执行采集任务（按优先级排序）
        ↓
每个采集器返回 SessionData 列表
        ↓
为每个会话生成摘要（LLM 或规则提取）
        ↓
根据 --period 参数选择报告周期：
  ├─ daily  → 生成日报（JSON + Markdown）
  ├─ weekly → 汇总本周日报，生成周报
  ├─ monthly→ 汇总本月日报/周报，生成月报
  └─ yearly → 汇总本年月报/周报，生成年报
        ↓
输出到指定目录
```

### 3.2 多级汇总架构

```
原始会话采集 → 单会话摘要 → 日报汇总 → 周报汇总 → 月报汇总 → 年报汇总
     ↑              ↑           ↑           ↑           ↑           ↑
  collectors   summarizer   generator   generator   generator   generator
```

- 每一级都基于上一级数据重新汇总提炼
- 支持逐级下钻查看详情
- 保存所有层级的汇总结果

## 四、插件化架构设计

### 4.1 采集器基类

```python
class BaseCollector(ABC):
    """采集器基类"""
    
    # 采集器元数据
    name: str                    # 采集器名称
    version: str                 # 版本号
    priority: int                # 优先级（越小越先执行）
    
    @abstractmethod
    def collect(self, target_date: datetime) -> List[SessionData]:
        """采集指定日期的会话数据"""
        pass
    
    def validate(self) -> bool:
        """验证采集器是否可用"""
        pass
    
    def get_data_path(self) -> Path:
        """获取数据源路径"""
        pass
```

### 4.2 注册机制

```python
# 采集器注册表
_COLLECTORS: Dict[str, Type[BaseCollector]] = {}

def register_collector(cls):
    """装饰器：注册采集器"""
    _COLLECTORS[cls.name] = cls
    return cls

def get_all_collectors() -> List[BaseCollector]:
    """获取所有已注册的采集器实例"""
    return [cls() for cls in _COLLECTORS.values()]
```

## 五、错误处理设计

### 5.1 采集器隔离

每个采集器独立运行，失败不影响其他采集器：

```python
def safe_collect(collector: BaseCollector, target_date: datetime) -> List[SessionData]:
    """安全执行采集任务"""
    try:
        return collector.collect(target_date)
    except Exception as e:
        logger.error(f"采集器 {collector.name} 失败: {e}")
        return []
```

### 5.2 降级策略

- 采集器失败 → 跳过该采集器，继续其他
- 数据源不存在 → 记录日志，返回空列表
- 数据格式异常 → 记录日志，跳过该条数据

## 九、扩展性设计

步骤：

2. 继承 `BaseCollector`

步骤：
